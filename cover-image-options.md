# 菜谱封面图生成 — 技术路线分析

## 需求

为每道菜谱自动生成封面图，用于卡片展示、分享卡片、转盘扇区背景。

## 路线对比

### 路线一：AI 图片生成 API（推荐）

| 方案 | 代表服务 | 成本 | 质量 | 速度 |
|------|----------|------|------|------|
| DALL·E / Midjourney API | OpenAI | $0.04/张 | ⭐⭐⭐⭐⭐ | 5-15s |
| Stable Diffusion 自部署 | SD WebUI | 显卡成本 | ⭐⭐⭐⭐ | 3-10s |
| 国内文生图 API | 通义万相/文心一格 | ¥0.02-0.1/张 | ⭐⭐⭐⭐ | 2-8s |

**适用场景**：菜谱数据量大，追求高质量，预算充足。

**实现思路**：写脚本遍历 recipes_full.json，用菜名+标签构造 prompt，调用 API 批量生成，图片存 CDN/OSS。

**优点**：质量最高，风格可控（可统一 prompt 模板）  
**缺点**：需要付费，批量生成 300 张 ≈ $12-15

---

### 路线二：Unsplash / Pexels 搜索 API（免费）

| 方案 | 免费额度 | 质量 |
|------|----------|------|
| Unsplash API | 5000次/小时 | ⭐⭐⭐⭐ |
| Pexels API | 200次/小时 | ⭐⭐⭐ |
| Pixabay API | 100次/分钟 | ⭐⭐⭐ |

**实现思路**：用菜名关键词搜索食物图片，取第一张做封面。

**优点**：免费，图片真实  
**缺点**：中文菜名搜不到（如"麻婆豆腐"→ 需翻译成"Mapo Tofu"），结果不可控，可能没有匹配图片

---

### 路线三：纯 CSS 渐变 + 图标（已有方案，零成本）

**当前项目已有此能力**：Card 卡片本身就是纯色温背景 + 文字排版，不依赖图片。

**增强方向**：
- 每种 subcategory 分配固定渐变色（牛肉=红褐，海鲜=蓝色，素菜=绿色）
- 卡片中央放 emoji 或 SVG 图标
- 转盘扇区用渐变色+文字（当前实现）

**优点**：零成本、零依赖、加载瞬间、风格统一  
**缺点**：没有真实食物照片的冲击力

---

### 路线四：Canvas 生成封面（折中方案）

用 Canvas 在客户端动态生成封面图：

- 背景：渐变色或纹理
- 前景：菜名 + 标签 + 难度星级
- 可加简单插画元素（筷子、锅、食材图标）

**实现思路**：在卡片渲染时画一个隐藏 Canvas，导出为 data URL 作为背景图，也可以缓存到 localStorage。

**优点**：零成本、离线可用、可统一风格  
**缺点**：不是真实照片，设计感取决于 Canvas 绘制能力

---

### 路线五：emoji/CSS 食物插画（最轻量）

每种菜谱根据 subcategory 或 tags 匹配一个代表性 emoji 做大图背景：

| 标签 | emoji |
|------|-------|
| 猪肉 | 🥩🐖 |
| 牛肉 | 🥩🐂 |
| 鸡肉 | 🍗🐔 |
| 海鲜 | 🦐🐟 |
| 面食 | 🍜🥟 |
| 米饭 | 🍚🍛 |
| 素菜 | 🥬🥦 |
| 豆腐 | 🧈🫘 |
| 蛋类 | 🍳🥚 |

CSS 将 emoji 放大到 80-120px 做卡片装饰背景，半透明叠加。

**优点**：零成本、零依赖、不占带宽、有趣  
**缺点**：不是食物照片

---

## 推荐组合方案

```
阶段一（立即可做）：路线三 + 路线五
  → 分类渐变色 + emoji 大图标，零成本，瞬间上线

阶段二（有预算）：路线一
  → 用 AI API 批量生成，替换掉 emoji 占位

阶段三（优化）：Canvas 生成分享卡
  → 抽到菜后生成带菜名+封面图的分享卡片
```

| 阶段 | 成本 | 效果 | 实施难度 |
|------|------|------|----------|
| 一 | 零 | 统一风格，有趣 | 低（1-2小时） |
| 二 | $12-15 | 高质量食物照片 | 中（API对接+批量脚本） |
| 三 | 零 | 社交传播力强 | 低（复用现有Canvas） |
