# 我用纯前端写了个「私房菜谱」网站：306道菜随便搜，还能用转盘抽奖决定今天吃什么

---

## 痛点：每天最难的决策不是写代码，是"吃什么"

作为一个程序员，我每天面对的灵魂拷问不是"这个bug怎么修"，而是——

**"晚上吃什么？"**

打开外卖App刷半小时，最后还是点了那家黄焖鸡。冰箱里明明有食材，但就是想不起来能做什么菜。

于是我做了一个决定：**把自己收集的300多道菜谱整理成一个网站，搜一搜就能找到想吃的，实在纠结就交给转盘抽奖。**

---

## 这个网站能做什么？

### 1. 智能搜索——输入"牛肉"或"下饭"，秒出结果

基于 Fuse.js 模糊搜索，支持按菜名、食材、口味、标签搜索。搜"牛肉"能出来红烧牛肉、水煮牛肉、滑蛋牛肉饭；搜"快手"能出来20分钟以内的菜。

**左侧分类树**按「家常菜/主食」分类，子分类精确到食材：猪肉、鸡肉、牛肉、海鲜、素菜、面食……点一下就筛选。

### 2. 菜谱详情——每一步都写得清清楚楚

点开任意一道菜，弹出完整详情：

- 食材清单（名称、用量、备注）
- 分阶段步骤（准备→炒制→收汁，每一步都有编号）
- 烹饪技巧（关键要点，不踩坑）
- 营养分析（热量、蛋白、碳水）

比如滑蛋鸡腿饭，从腌鸡腿到滑蛋到组装，5个阶段11步，每一步都写了为什么这么做。

### 3. 🎰 转盘抽奖——让命运决定今天吃什么

这是我最喜欢的功能，也是写代码最开心的部分。

**标签抽奖**：选一个标签（比如「牛肉」），系统随机抽10道牛肉菜放到转盘上，点击"开始抽奖"，转盘旋转4-5秒后自然减速停下——指针指向哪道，今天就吃哪道。

**自定义抽奖**：你也可以输入自己的选项，比如：
```
黄焖鸡
麻辣香锅
螺蛳粉
兰州拉面
```
生成转盘，一样能抽。

**技术细节**：转盘用 Canvas 绘制，每个扇区不同颜色，文字方向自动适配——上半部正常读，下半部翻转180度，确保始终可读。旋转动画用 `requestAnimationFrame` + cubic ease-out 缓动，模拟物理减速，丝滑流畅。

## 技术栈

整个项目**纯前端**，没有后端，部署在 Vercel 上：

| 技术 | 用途 |
|------|------|
| HTML5 / CSS3 / JS(ES6+) | 原生三件套，零框架 |
| Fuse.js | 模糊搜索 |
| Canvas API | 转盘绘制 |
| localStorage | 数据持久化 |
| Vercel | 免费部署 |

代码**模块化拆分**：`index.html` 纯结构、`styles.css` 全部样式、`app.js` 全部逻辑，清晰好维护。

## 开源 & 数据管理

项目已开源在 GitHub，附带一个 **Python 菜谱解析脚本** `parse_recipe.py`：

```bash
# 把菜谱文字一键转成JSON，追加到菜谱库
python3 parse_recipe.py 番茄炒蛋.txt \
  --subcategory 素菜 \
  --tags 番茄,鸡蛋,快手,下饭 \
  --time 15
```

脚本自动识别菜名、食材表格、步骤分段、技巧、营养信息，加上确认机制，不会误写数据。
